Instructions to be read to the participant:

that you may have. Throughout the experiment, do your best to avoid slouching

After having understood these instructions, you may proceed with the first
series of tasks. This series will last for 15 minutes at which point the
researcher will come to announce its end. You will be given a short break while
the researcher finishes the setup for the second series of tasks. You will then
be asked to begin the next series of tasks.

Each task within a series of tasks should take approximately 5 minutes. At the
end of the experiment, you will be required to sign a consent form and retrieve
your compensation.

DISCLAIMER: Before each series of tasks, the researcher will help you with
calibrating the eye-tracker.

Instructions for TASK SERIES 1:

In TASK SERIES 1, you will be solving coding problems with the help of GitHub
Copilot. If you have any questions, you must ask Copilot as the researcher will
not be available.

- Use CTRL+Click to explore functions.
- Use "python -m glances" in the terminal to run Glances GUI (errors indicate your solution is wrong).
- Use CTRL+X exit Glances GUI and return to your terminal.

1. File: glances/processes.py
   Description: In the update_processcount method, ensure that process statuses are
       correctly counted by handling edge cases like missing or
       invalid status values in the process list. Modify the lambda
       function used in filtering to account for such cases.
   Hint: Should we be verifying object identity (same object in memory) or
       object equality (same object in value)? Which operator may be
       causing this issue?

2. File: glances/processes.py
   Description: Enhance the fallback logic in the get_io_counters method to include a
       timestamp whenever the IO counters are initialized with default
       values. Use the time module to add a timestamp field in the
       proc['io_counters'] list that records when the fallback logic
       was triggered.
   Hint: Consider where in the else block the timestamp would fit best. Look
       into the Python time module and its time() function.

3. File: glances/server.py
   Description: Add a new method serve_limited to the GlancesXMLRPCServer class. This
       method allows the server to handle a limited number of requests
       before stopping. Use a request counter to track the number of
       handled requests and call the end method when the limit is
       reached. The method should log both the start of the limited
       server operation and when it stops due to reaching the limit.
   Hint: How can you integrate a simple counter to track the number of
       iterations in a loop? Consider using an existing method to
       terminate the server gracefully once the count exceeds the
       given limit.


Once you have finished the tasks, you may inform the researcher, wait for them
